## 1. アプリの全体構成

```
【管理者】                          【参加者】
admin.html                         index.html
（ローカルまたはGitHub Pages）      （GitHub Pages で公開）
    │                                   │
    │ ① 結果を入力                      │ ランキング・結果を閲覧
    │ ② JSONダウンロード                │
    ▼                                   ▼
data/data.json  ──── GitHubにpush ──▶  data/data.json
（データファイル）                     （自動で読み込まれる）
```

### ポイント
- **管理者だけ** が `admin.html` を使って結果を入力する
- **参加者全員** は GitHub Pages の URL を開くだけで最新ランキングを確認できる
- データの更新は「JSONダウンロード → ファイル上書き → GitHub push」の3ステップ
- サーバー不要・完全無料で運用できる

## 3. 初回セットアップ（GitHub Pages公開まで）

### ステップ① GitHubにファイルをアップロードする

1. `https://github.com/ketu-yakuman/sanma-calc` を開く
2. 「**Add file**」→「**Upload files**」をクリック
3. フォルダの中身をすべてドラッグ＆ドロップ
4. 「**Commit changes**」をクリック

### ステップ② GitHub Pages を有効にする

1. リポジトリの「**Settings**」→「**Pages**」
2. Source：`Deploy from a branch`
3. Branch：`main` / `/ (root)`
4. 「**Save**」

公開URL：`https://ketu-yakuman.github.io/sanma-calc/`

### ステップ③ 参加者の選択国を登録する

1. `admin.html` をブラウザで開く（またはGitHub Pages のURLで開く）
2. 「📂 data.json を読み込む」
3. 「👥 参加者・選択国」タブで6か国を入力
4. 「💾 JSONをダウンロード」→ GitHubにpush

## 4. 管理者の操作方法

### 基本的な操作の流れ

```
① admin.html を開く
② 「📂 読込」で data.json を読み込む  ← 毎回必ず最初に行う
③ 各タブで入力・編集
④ 「💾 保存」でJSONをダウンロード
⑤ data/data.json に上書き保存
⑥ GitHub に push
```

> ⚠️ **必ず②の読み込みから始めること。**
> 読み込まずに入力すると、前回の変更が消えてしまう。

### iPhoneでの操作

```
① Safari で以下のURLを開く
   https://ketu-yakuman.github.io/sanma-calc/admin.html

② GitHubから最新のdata.jsonをダウンロード
   GitHub → data/data.json → Raw → 共有 → "ファイル"に保存

③ 「📂 読込」でダウンロードしたdata.jsonを選択

④ 結果を入力 → 「💾 保存」

⑤ GitHubにdata.jsonをアップロード
```

### タブ①：👥 参加者・選択国
- 参加者名の変更・選択国（6か国）の入力
- 国はドロップダウンから選択（あいうえお順・48か国）

### タブ②：⚽ グループ試合
- スコアを入力して「✓ 試合済」にチェック
- 未来の試合は青ライン、試合済みは緑ラインで色分け
- 日時順に並び替え表示（未試合が上、試合済みが下）

### タブ③：🎯 決勝トーナメント
- グループ順位表が自動表示
- 1位・2位（24チーム）は「✅ ベスト32に反映する」で自動反映
- 3位通過の8チームは手動で選択
- ベスト16以降も順次入力

### タブ④：🌟 ボーナス・ペナルティ
- 大会得点王・MVPの所属国をドロップダウンで選択（＋10点）
- 優勝国ボーナスは優勝国設定で自動加算（＋10点）
- **3位決定戦の勝者国をドロップダウンで選択（＋10点）**
- 大会最多レッドカード国をドロップダウンで選択（－10点）
- 大会最多失点国をドロップダウンで選択（－10点）

## 7. 得点ルールの詳細

### グループリーグ（選択国の試合ごとに加算）

| 結果 | 点数 |
|------|------|
| 勝利 | ＋3点 |
| 引き分け | ＋1点 |
| 敗戦 | 0点 |

### 決勝トーナメント（ラウンドが上がるたびに累積加算）

| ラウンド | 点数 | 累積最大 |
|----------|------|---------|
| ベスト32進出 | ＋3点 | 3点 |
| ベスト16進出 | ＋5点 | 8点 |
| ベスト8進出 | ＋8点 | 16点 |
| ベスト4進出 | ＋12点 | 28点 |
| 決勝進出 | ＋15点 | 43点 |
| 優勝 | ＋25点 | 68点 |

### ボーナス

| 条件 | 点数 |
|------|------|
| 大会得点王の所属国を選んでいる | ＋10点 |
| 大会MVPの所属国を選んでいる | ＋10点 |
| 優勝国を選んでいる | ＋10点 |
| **3位決定戦の勝者国を選んでいる** | **＋10点** |

> ボーナスは重複適用OK。優勝国の選手が得点王＆MVPなら最大＋30点
> ボーナス点はすべて2倍補正の対象外

### ⚠️ 逆ボーナス（ペナルティ）

| 条件 | 点数 |
|------|------|
| 大会最多レッドカード国を選んでいる | **－10点** |
| 大会最多失点国を選んでいる | **－10点** |

> ペナルティは2倍補正の対象外（そのまま－10点）
> 同一国が両方に該当した場合は最大－20点

### ⭐ FIFAランキングによる得点倍率（追加ルール）

選択した国のFIFAランキングに応じて、得点に倍率がかかります。

| ランキング | 倍率 | 対象国 |
|---|---|---|
| 1〜49位 | ×1（通常） | アルゼンチン、フランス、日本など |
| **50位以下** | **×2（2倍）** | ウズベキスタン、カタール、イラク、南アフリカ、サウジアラビア、ヨルダン、ボスニア・ヘルツェゴビナ、カーボベルデ、ガーナ、ハイチ、キュラソー、ニュージーランド |

> ⚠️ ボーナス点（得点王・MVP・優勝国）は2倍の対象外

**例：ハイチ（81位）がグループリーグで1勝1分1敗の場合**
```
通常得点：3 + 1 + 0 = 4点
2倍適用後：4 × 2 = 8点
```

### 選択国のルール制約
- FIFAランキング10位以内の国は **2か国まで**
- FIFAランキング50位以下の国を **1か国以上** 含めること

### 同点の場合の順位決定

1. 保有国の総得失点差（大きい方が上位）
2. 優勝国を保有しているか
3. じゃんけん

> ※得失点差はアプリでは自動計算されないため手動で確認

## 9. よくある質問・トラブル対応

### Q. GitHubにpushしたのに公開ページが更新されない
→ GitHub Pages の反映には **1〜3分** かかることがある。少し待ってからリロード。

### Q. admin.html を開いたらデータが空になっている
→ 毎回「📂 読込」から最新ファイルを読み込む必要がある。

### Q. iPhoneのファイルアプリからadmin.htmlを開いてもボタンが動かない
→ Safariで以下のURLを直接開いてください。
`https://ketu-yakuman.github.io/sanma-calc/admin.html`

### Q. 誤って古いdata.jsonをpushしてしまった
→ GitHubのリポジトリ画面でファイルを直接アップロードし直す。

### Q. 参加者が選択国を変更したい
→ 「提出後の変更は不可」のルールを適用。変更しない。

### Q. 同じ国を複数の参加者が選んでいる場合
→ 問題なく動作する。それぞれの参加者の得点に同じ国のポイントが加算される。
