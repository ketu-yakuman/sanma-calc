## 1. アプリの全体構成

```
【管理者】                          【参加者】
admin.html                         index.html
（ローカルで開く）                   （GitHub Pages で公開）
    │                                   │
    │ ① 結果を入力                      │ ランキング・結果を閲覧
    │ ② JSONダウンロード                │
    ▼                                   ▼
data/data.json  ──── GitHubにpush ──▶  data/data.json
（データファイル）                     （自動で読み込まれる）
```

### ポイント
- **管理者だけ** が `admin.html` を使って結果を入力する
- **参加者全員** は GitHub Pages の URL を開くだけで最新ランキングを確認できる
- データの更新は「JSONダウンロード → ファイル上書き → GitHub push」の3ステップ
- サーバー不要・完全無料で運用できる

## 2. ファイル構成と役割

```
wc2026/
├── index.html          ★ 参加者が見る公開ページ
├── admin.html          ★ 管理者が使う入力画面（公開しない）
├── js/
```

```json
{

    "round_of_16": [],

  },
  "bonuses": {                // ボーナス（得点王・MVP の所属国）
    "topScorer": "",
    "mvp": ""
  }
}
```

## 3. 初回セットアップ（GitHub Pages公開まで）

### ステップ① GitHubリポジトリを作成する

1. [https://github.com](https://github.com) にアクセスしてログイン
2. 右上の「＋」→「New repository」をクリック
3. 以下のように設定する
   - Repository name: `wc2026`（任意）
   - Public を選択（GitHub Pages に必要）
   - 「Create repository」をクリック

### ステップ② ファイルをアップロードする

**方法A：GitHub のWeb画面からアップロード（簡単）**

1. 作成したリポジトリのページを開く
2. 「uploading an existing file」をクリック
3. `wc2026` フォルダの中身をすべてドラッグ＆ドロップ
   - `index.html`
   - `admin.html`
   - `js/scoring.js`
   - `data/data.json`
   - `README.md`
   - ※フォルダごとまとめてドロップでOK
4. 「Commit changes」をクリック

**方法B：Git コマンドを使う場合**

```bash
cd wc2026フォルダのパス
git init
git add .
git commit -m "初期設定"
git remote add origin https://github.com/あなたのID/wc2026.git
git push -u origin main
```

### ステップ③ GitHub Pages を有効にする

1. リポジトリの「**Settings**」タブをクリック
2. 左メニューの「**Pages**」をクリック
3. 「Source」のドロップダウンを「**Deploy from a branch**」に変更
4. Branch を「**main**」／「**/ (root)**」に設定
5. 「**Save**」をクリック
6. 数分待つと以下のURLで公開される

```
https://あなたのGitHubID.github.io/wc2026/
```

> ✅ このURLを参加者8名に共有すればOK

### ステップ④ 参加者の選択国を登録する

1. `admin.html` をブラウザで直接開く（ダブルクリックでOK）
2. 「📂 data.json を読み込む」ボタンをクリック
3. `data/data.json` を選択して読み込む
4. 「👥 参加者・選択国」タブで各参加者の6か国を選択
5. 「💾 JSONをダウンロード」をクリック
6. ダウンロードされた `data.json` を `data/` フォルダに上書き保存
7. GitHubにpush（またはWebアップロード）

## 4. 管理者の操作方法

### admin.html の開き方

- `admin.html` をブラウザで**直接開く**（ファイルをダブルクリック）
- GitHub Pages には公開されているが、URLを知らなければ参加者はアクセスしない
- 特に機密性が高くない場合はそのままでOK

### 基本的な操作の流れ

```
① admin.html を開く
② 「📂 data.json を読み込む」で現在のデータを読み込む  ← 毎回必ず最初に行う
③ 各タブで入力・編集
④ 「💾 JSONをダウンロード」
⑤ data/data.json に上書き保存
⑥ GitHub に push
```

> ⚠️ **必ず②の読み込みから始めること。**
> 読み込まずに入力すると、前回の変更が消えてしまう。

---

### タブ①：👥 参加者・選択国

| 操作 | 説明 |
|------|------|
| 参加者名の変更 | テキストボックスで直接編集 |
| 選択国の入力 | 各参加者の6つのドロップダウンから国を選ぶ |

- 大会開幕前に一度だけ入力する
- 選択国は**48か国のドロップダウン**から選ぶ（入力ミス防止）

---

### タブ②：⚽ グループ試合

#### 試合を追加する（通常は不要）
- 72試合分はあらかじめ登録済み
- 追加が必要な場合のみ「ホーム」「アウェイ」「グループ」を入力して追加

#### 試合結果を入力する
1. 試合一覧から該当の試合を探す
2. スコアを左右の入力欄に入力（例：2 : 1）
3. 「✓ 試合済」チェックボックスをオンにする
4. ✓がオンになった試合のみ得点計算に反映される

> 💡 複数試合をまとめて入力してからJSONを一度だけダウンロードすればOK

---

### タブ③：🎯 決勝トーナメント

決勝トーナメントの各ラウンドで進出した国を選んでいく。

| ラウンド | 入力する国数 | 対応得点 |
|----------|------------|---------|
| ベスト32進出 | 32か国 | ＋3点 |
| ベスト16進出 | 16か国 | ＋5点 |
| ベスト8（準々決勝） | 8か国 | ＋8点 |
| ベスト4（準決勝） | 4か国 | ＋12点 |
| 決勝進出 | 2か国 | ＋15点 |
| 🏆 優勝国 | 1か国 | ＋25点（＋ボーナス10点） |

- 各ラウンドで進出が確定した国からドロップダウンで選択していく
- **累積入力**なので、ベスト32の32か国を入れたあとベスト16の16か国を追加で入れる

---

### タブ④：🌟 ボーナス

| 項目 | 説明 |
|------|------|
| 大会得点王の所属国 | その国を選択している参加者に＋10点 |
| 大会MVPの所属国 | その国を選択している参加者に＋10点 |

- 大会終了後に入力する
- 優勝国ボーナス（＋10点）は**優勝国を設定すると自動で加算**される（入力不要）

## 5. 参加者の見方

GitHub Pages のURL（例：`https://xxx.github.io/wc2026/`）を開くと以下が確認できる。

### 🏆 ランキングタブ
- 現在の順位と得点を表示
- 各参加者の選択国ごとの内訳点数も表示
- 1〜3位には賞金額のバッジが表示される

### 🌍 選択国タブ
- 参加者ごとの選択6か国を一覧表示
- すでに得点を獲得している国はハイライト表示される

### ⚽ 試合結果タブ
- グループリーグの全試合結果を確認できる
- 未試合は「vs」、試合済みはスコアが表示される

### 🎯 決勝Tタブ
- 決勝トーナメントの各ラウンド進出国を確認できる
- 優勝国・ボーナス情報も表示される

## 7. 得点ルールの詳細

### グループリーグ（選択国の試合ごとに加算）

| 結果 | 点数 |
|------|------|
| 勝利 | ＋3点 |
| 引き分け | ＋1点 |
| 敗戦 | 0点 |

> 例：日本を選んでいて、日本がグループリーグで2勝1敗なら 3+3+0 = **6点**

### 決勝トーナメント（ラウンドが上がるたびに累積加算）

| ラウンド | 点数 | 累積最大 |
|----------|------|---------|
| ベスト32進出 | ＋3点 | 3点 |
| ベスト16進出 | ＋5点 | 8点 |
| ベスト8進出 | ＋8点 | 16点 |
| ベスト4進出 | ＋12点 | 28点 |
| 決勝進出 | ＋15点 | 43点 |
| 優勝 | ＋25点 | 68点 |

> 例：日本がベスト8で敗退した場合 → 3+5+8 = **16点**（決勝Tだけで）

### ボーナス

| 条件 | 点数 |
|------|------|
| 大会得点王の所属国を選んでいる | ＋10点 |
| 大会MVPの所属国を選んでいる | ＋10点 |
| 優勝国を選んでいる | ＋10点 |

> ボーナスは重複適用OK。優勝国の選手が得点王＆MVPなら最大＋30点

### 同点の場合の順位決定

1. 保有国の総得失点差（大きい方が上位）
2. 優勝国を保有しているか
3. じゃんけん

> ※得失点差はアプリでは自動計算されない。手動で確認すること。

## 8. よくある質問・トラブル対応

### Q. GitHubにpushしたのに公開ページが更新されない
→ GitHub Pages の反映には **1〜3分** かかることがある。少し待ってからリロード。

### Q. admin.html を開いたらデータが空になっている
→ 毎回「📂 data.json を読み込む」から最新ファイルを読み込む必要がある。ブラウザを閉じると前回のデータは消える。

### Q. 誤って古いdata.jsonをpushしてしまった
→ GitHubのリポジトリ画面でファイルを直接編集するか、正しいファイルを再pushする。

### Q. 参加者が選択国を変更したい
→ 「提出後の変更は不可」のルールを適用。変更しない。

### Q. 同じ国を複数の参加者が選んでいる場合
→ 問題なく動作する。それぞれの参加者の得点に同じ国のポイントが加算される。

### Q. 公開ページのURLを変えたい
→ GitHubのリポジトリ名を変えればURLが変わる（Settings → General → Repository name）
